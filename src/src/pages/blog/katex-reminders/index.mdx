## Gotchas on escape characters & unicode
### For Katex
While using Katex in the browser, you may encounter issues with Unicode and character encoding.
This one <FakeUnicode /> has a particularly bad behaviour as it makes JS trying to interpret
the following characters as an Unicode. To prevent this interpretation, simply use:

```js
String.raw`...Any raw string that you want...`
```

More information on [MSN web docs](https://developer.mozilla.org/fr/docs/Web/JavaScript/Reference/Objets_globaux/String/raw).

### Fox MDX
Like for Katex, some escape character may crash your MDX display. With MDX capability
to call any React component on demand, simply create a special component that will get
rid of the interpretation, like in this example.
