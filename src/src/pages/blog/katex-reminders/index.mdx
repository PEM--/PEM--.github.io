## Existence & condition

<Existence />

> `~` is handy character in Katex for injecting a space into your equation for readability.

```katex
\forall~x \in \reals, \exist ~a ~and~ b \in \N ~where~
x = \begin{cases}
  a < x \\
  x < b
\end{cases}
```

## Operations on intervals

<OperationOnInterval />

> For other big operator, just replace `\sum` with your operator of choice.
> Here's a list of commonly used ones:
> * `\sum` <Katex formulae='\sum' />
> * `\prod` <Katex formulae='\prod' />
> * `\int` <Katex formulae='\int' />
> * `\iint` <Katex formulae='\iint' />

```katex
\begin{aligned}
  S_n &= \sum_{n\in\N} a^n \\
      &= 1 + a + a^2 + \dots + a^n \\
      &= \cfrac{1 - a^{n+1}}{1 - a}
\end{aligned}
```

## Gotchas on escape characters & unicode
### For Katex
While using Katex in the browser, you may encounter issues with Unicode and character encoding.
This one <FakeUnicode /> has a particularly bad behaviour <Emoji text=':scream:.' /> It makes
JS trying to interpret the following characters as an Unicode character. To prevent this interpretation,
simply use:

```js
String.raw`...Any raw string that you want...`
```

More information on [MSN web docs](https://developer.mozilla.org/fr/docs/Web/JavaScript/Reference/Objets_globaux/String/raw).

### Fox MDX
Like for Katex, some escape characters may crash your MDX rendering. With MDX ability
to call any React component on demand, simply create a special component that will get
rid of the pesky interpretation, like in this [example](https://github.com/PEM--/PEM--.github.io/blob/master/src/src/pages/blog/katex-reminders/FakeUnicode.js).
